# 宝塔面板部署指南 — 尖塔卡牌

## 需要上传的文件（总共 2 个）

| # | 文件 | 生成方式 |
|---|------|----------|
| 1 | `text-fighting-game-1.0.0.jar` | 本地执行 `cd backend && mvn package -DskipTests` |
| 2 | `dist/` 目录（整个文件夹） | 本地执行 `cd frontend && npm run build` |

> **简明流程**：后端 JAR → 宝塔 Java 项目 | 前端 dist → 宝塔网站目录 + Nginx 反向代理

---

## 一、部署后端（JAR 包）

### 1. 宝塔安装 Java 环境

宝塔面板 → 软件商店 → 搜索安装 **Java 项目一键部署**（会自动安装 JDK）

如未找到，手动安装：
```bash
# Ubuntu/Debian
apt install openjdk-17-jdk -y

# CentOS
yum install java-17-openjdk -y

# 验证
java -version
```

### 2. 上传 JAR 包

将 `text-fighting-game-1.0.0.jar` 上传到：
```
/www/wwwroot/card-game/backend/
```

### 3. 创建 Java 项目

宝塔 → 网站 → Java 项目 → 添加 Java 项目：

| 配置项 | 值 |
|--------|-----|
| 项目名称 | card-game-backend |
| 项目路径 | `/www/wwwroot/card-game/backend` |
| JDK 版本 | JDK 17 |
| 端口 | `8080` |
| 执行命令 | `java -jar text-fighting-game-1.0.0.jar` |

提交后确认状态为 **运行中**。

### 4. 验证后端

```bash
curl http://localhost:8080/api/game/state
# 应返回 JSON: {"phase":"welcome","ok":true}
```

---

## 二、部署前端（静态文件）

### 1. 上传 dist 目录

将本地 `frontend/dist/` 目录中的所有文件上传到：
```
/www/wwwroot/card-game/frontend/
```

结构应为：
```
/www/wwwroot/card-game/frontend/
├── index.html
└── assets/
    ├── index-xxx.css
    └── index-xxx.js
```

### 2. 创建网站

宝塔 → 网站 → 添加站点：

| 配置项 | 值 |
|--------|-----|
| 域名 | 你的域名 或 服务器IP |
| 根目录 | `/www/wwwroot/card-game/frontend` |

### 3. 配置 Nginx

站点 → 设置 → 配置文件，写入：

```nginx
server {
    listen 80;
    server_name 你的域名;   # ← 替换这里

    # 前端静态文件
    location / {
        root /www/wwwroot/card-game/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

保存后宝塔会自动重载 Nginx。

## 三、验证

浏览器访问你的域名 → 应该看到「尖塔卡牌」欢迎页面 → 输入名字 → 开始游戏。

如果能正常进入地图、打牌战斗，部署成功。

## 四、常见问题

### Q1: 访问页面提示"连接服务器失败"

1. 检查后端 Java 项目是否运行（`netstat -tlnp | grep 8080`）
2. 检查 Nginx `/api/` 代理配置是否正确
3. 查看后端日志：宝塔 Java 项目 → 日志

### Q2: 游戏状态丢失

游戏通过 HttpSession 维护状态，确保不要关闭浏览器 Cookie。每个浏览器会话独立。

### Q3: 修改后端端口

1. 编辑 `application.properties` → `server.port=9090`
2. 重新 `mvn package` 
3. 同步修改 Nginx 中 `proxy_pass http://127.0.0.1:9090`

### Q4: 只上传 dist 目录不就行了吗？为什么还需要单独部署后端？

前端 dist 是纯静态文件（HTML+CSS+JS），不包含游戏逻辑。游戏的核心计算（战斗、卡牌效果、敌人AI、随机数等）全部在后端 JAR 中运行。前端通过 `/api/` 请求调用后端接口，后端返回游戏状态。
